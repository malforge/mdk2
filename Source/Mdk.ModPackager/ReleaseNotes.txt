v.2.2.7
   - Source-defined attribute applications and declarations now can be used during compile-time and automatically trimmed from packed mods.

v.2.2.5
   - Mods can be packed into a different output folder per git branch via [mdk-branch:<branch>] sections in mdk.ini, so you can publish a separate alpha/test Workshop item from a branch without touching your released mod. The folder name comes from a 'pattern' supporting the $MDK_PROJECT$ and $MDK_BRANCH$ macros.
   - A branch build watermarks its thumbnail by default with a translucent red stamp (set watermark=false to disable; watermarktext to customize), so alpha/test items are easy to spot in the Workshop.
   - Branch sections in mdk.local.ini take precedence over mdk.ini, matching every other setting.

v.2.2.5-pre.2
   - [mdk-branch:<branch>] sections now stamp a translucent red watermark onto the deployed thumbnail by default (set watermark=false to disable; watermarktext to customize) so alpha/test builds are easy to spot in the Workshop.
   - Branch sections in mdk.local.ini now correctly take precedence over mdk.ini, matching every other setting.

v.2.2.5-pre.1
   - Mods can now be packed into a different output folder per git branch, via [mdk-branch:<branch>] sections in mdk.ini. This lets you publish a separate alpha/test Workshop item from a branch without touching your released mod; the folder name is set by a 'pattern' that supports the $MDK_PROJECT$ and $MDK_BRANCH$ macros. Branches with no matching section pack to the project name as before.

v.2.2.4
   - `#if` directives now use the project's actual conditional compilation symbols instead of the build configuration name.
   - Mod projects are now automatically registered with the MDK Hub after a successful pack, the same way Programmable Block scripts already were.
   - The "MDK Hub Not Found" page now includes diagnostics to help identify stale or misconfigured Hub installations.
